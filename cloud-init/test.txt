I think the arguments to do that would be something like
-net user,hostfwd=tcp::2222-:22

(or -net user,hostfwd=tcp:127.0.0.1:2222-:22
if you only wish to allow connections from the host itself),
where "2222" is the port to forward on the host side,
and "22" (well-known ssh port) is the port to which to forward on the guest side.
Once you start qemu with this argument, you should be able to ssh to the guest with ssh -p 2222 (or whatever number you chose).

ssh ubuntu@localhost -p2222
